# Дизайн документ - Системы формирования команд для марафона
## Проблема
- Участники покидают марафон, потому что не находят способных людей или вообще никого не находят.
- Часто участники покидают марафон по причине "не сработался" из за работы с первыми встречными. 
- Психологический барьер, Многие участники в принципе не принимают попытки поиска из-за стеснения. 

*Я лично общался со способными студентами, которые покидали командные марафоны по причинам выше.*

## Решение, краткое описание
Новая система формирования команд и поиска участников по модели рекрутинговых платформ (пример hh.ru), где команда - работодатель, а участник марафона - соискатель. 

## Термины
- Пользователь - любой пользователь сайта
- Марафон - командное мероприятие/хаккатон, созданное пользователем
- Организатор - создатель марафона или пользователь, которого пригласили к управлению марафоном
- Участник - пользователь, который присоединился к марафону

## Система
### Регистрация на сайте
Пользователь имеет возможность авторизоваться через почту и код с почты. Код действителен 10 минут. 

### Создание марафона
Организаторы марафона регистрируются на сайте и указывают 
- Название марафона 
- Краткое название (латинские буквы, цифры, "-_", регистр не учитывается, до 16 символов)
- Допустимый диапазон количества участников в команде. 

Сайт для марафона будет доступен по адресу `/{краткое_название}/`

### Участие в марафоне
- Пользователь имеет возможность присоединиться к марафону или покинуть его по адресу `/{краткое_название}/`.

### Управление марафоном
**Организатор имеет возможность**
- Измененить любую информацию о марафоне
- Исключить участника без возможности возвращения в марафон (Бан)
- Временно отстранить участника написав причину, участник не будет отображаться в списке участников
- Временно отстранить команду написав причину, команда не будет отображаться в списке команд
- Удалить команду

### Создание команды
При регистрации команды надо будет указать некоторую информацию. Также все данные о команде кроме названия и состава можно скрыть по желанию.

**Основное**
- Название команды

**Управление командой**
- Тип менеджмента (Scrum, Kanban, Agile, Waterfall, Свободный стиль)
- Система принятия решений (Диктатура/Демократия), глобальные решения по составу команды, решает "тимлид" или большинство

**Тимлид**
- Создатель команды автоматически становится тимлидом
- Тимлид может передать роль другому участнику команды через запрос
- Если тимлид покидает команду, система принятия решений автоматически переходит в режим "Демократия"
- При смене системы принятия решений с "Демократия" на "Диктатура" в запросе указывается кандидат на роль тимлида

**Голосование (Демократия)**
- Решение принимается когда проголосовало большинство участников команды (>50%)

**Распад команды**
- Когда последний участник покидает команду, команда автоматически удаляется.

**Дополнительные материалы (опционально)**
- Жанр игры
- Краткое описание
- Питч проекта
- Дизайн-документ
- Ссылка на чат команды
- Ссылка на репозиторий

### Профиль участника
Каждый участник по желанию может заполнить информацию о себе для поиска команды и более удобного взаимодействия в будущем. Для отдельных марафонах отдельные профили, также есть возможность заполнять уже написанные данные о себе с другого марафона. 
- ФИО
- Уникальный Ник (латинские буквы, цифры, "-_", регистр не учитывается, до 16 символов)
- Роли (программист, дизайнер, геймдизайнер, звукорежиссёр, итд.)
- Технологии (Unity, Unreal, Godot, Figma, Photoshop, итд.)
- Описание: Портфолио, Навыки и технологии, Предпочтения

### Поиск участников
**Каждая команда может**
- Просматривать список всех свободных участников
- Фильтровать по критериям (роль, технологии) 
- Отправлять приглашения участникам
- Также можно указать о том что команда в поиске участника с кратким описанием. 

Любой участник команды может отправить запрос на приглашение участника или на открытие позиции. Большинство или тимлид (в зависимости от выбранной системы принятия решений) должны одобрить запрос. 

### Поиск команд
**Участники могут**
- Просматривать список всех команд и всю открытую информацию о каждой из команд
- Фильтровать по критериям (тип менеджмента, система принятия решения, жанр, открыто место на роль)
- Отправлять заявки в интересующие команды, прикладывать письмо к заявке. Заявка существует до её отмены участником или обработки командой.
- Принимать или отклонять приглашения от команд. Участник может иметь несколько активных приглашений одновременно. При вступлении в команду остальные приглашения автоматически становятся недействительными.
- Покинуть команду

### Управление командой
Для управления командой будут открываться запросы, которые должны подтвердить тим-лид или большинство (в зависимости от выбранной системы принятия решений).
- Изменение состава/открытых мест
- Изменения настроек которые задавались при регистрации
- Передача роли тимлида
- Изменение системы принятия решений (при смене на "Диктатура" указывается кандидат на тимлида)
